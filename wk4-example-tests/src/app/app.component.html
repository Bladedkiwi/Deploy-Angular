• ngClass adds the desired class if the following expression is true
  • ngStyle adds the desired style if the following expression is true
  • {{ odd }}
    • * indicates a structural directive. There's no star operator, angular translates it.
    • Angular translates the * into a ngTemplate essentially.
    • ngTemplate is an element that's not rendered like the template tag.
      • Allows for adding customizable elements that activate on events or conditions.
    • Creating a custom structural directive follows similar rules as an attribute directive.
      • The constructor should initialize the TemplateRef of any type and the ViewContainerRef
      • Then, with a set @Input method, you can alter the template to show or not via the createEmbeddedView method from the ViewContainerRef
  • {{ even }}
  • @Directive to create a custom directive to use/modify.
    • selector is required - either by element p or attribute style [p]
  • Directive Instances that are created need a reference to the element that it was created on via a parameter in the constructor method.
    • Use private to provide access to the element
  • Changing the exact element best done in ngOnInit
    • Altering through the DOM via nativeElement is not the best practice
    • Best practice is to manipulate the DOM through Renderer2 and its setStyle() method.
    • More on Renderer2
    • Easiest way to alter color is through @HostBinding with the 'style.backgroundColor'.
  • Remember to add the custom directive to the module file
  • No brackets are needed for the element for basic use like having the green highlight as a default
  • @HostListener convenient way to listen to events on the element like mouseenter or mouseleave
  • Adding @Input default/altered values to child component allows for access from the html file and easy changing.
